Getting Started

1.1  Logging In

  1. 1Open your browser and navigate to the application URL provided by your administrator.
  2. 2Enter your username (or email) and password.
  3. 3Click Sign In. You will be redirected to the Dashboard.
✅
If you forget your password, contact your system administrator to reset it.

1.2  Navigation Sidebar

The left sidebar is always visible and provides access to all main sections of the application.

  • Dashboard — Overview and quick stats
  • Items — Browse all pawned articles
  • Customers — Customer directory
  • Branches — Branch management
  • Audits — Audit sessions
  • Reports — Analytics & summaries
  • Activity Log — System event history

Section 2

Items — Pawned Articles

2.1  Browsing & Searching Items

Navigate to Items in the sidebar. Items are grouped by Ticket Number. Click any ticket row to expand and see individual items.

🔍 Search & Filter

  • Type in the search bar to find by ticket no, barcode, or customer name
  • Use the Status filter to show only Active or Released items
  • Results update as you type

📄 Ticket Groups

  • Each row shows the Ticket No, customer name, pawn date, item count, and total weight
  • Click the ▶ chevron (or the row) to expand individual items
  • Click customer name to go directly to their profile
✅
Use the CSV Import button to bulk-upload items from a spreadsheet. Download the template first to ensure correct column format.

2.2  Creating a New Item  Admin Only

  1. 1Click the + New Item button (top-right of Items page).
  2. 2Fill in the form fields (see table below).
  3. 3Click Save Item. A barcode is automatically generated.
FieldRequiredDescription
Customer✅ YesSearch and select existing customer
Ticket No✅ YesPawn ticket number (e.g. TK-001)
Pawn Date✅ YesDate the item was pawned
Item Type✅ Yesring, chain, bracelet, earrings, necklace, bangle, pendant, other
Net Weight (g)✅ YesNet weight of gold in grams (3 decimal places)
Gross Weight (g)NoTotal weight including stones/setting
Karatage (K)NoGold purity e.g. 22, 18, 14
RemarksNoFree text notes about the item

2.3  Editing an Item  Admin Only

Inline Edit (Items list page):

  1. 1Expand a ticket group and then expand an individual item.
  2. 2Click the ✏️ Edit button in the item row.
  3. 3Modify the fields and click 💾 Save.
  4. 4All changes are recorded in edit history.

Edit via Item Detail page:

  1. 1Click Full Detail on any item.
  2. 2Click Edit Item in the detail panel.
  3. 3Modify fields and Save.
🔒
Editing is disabled on items that are part of a finalized audit.
🏷️
Section 2 (cont.)

Item Detail & Barcode Printing

2.4  Item Detail Page

Click Full Detail on any item to open its dedicated page at /items/:id.

📋 Information Displayed

  • Barcode — large scannable barcode image
  • Customer — name, NIC, phone with link to profile
  • Item facts — type, net weight, gross weight, karatage, remarks
  • Dates — pawn date, registration date, release date (if released)
  • Status — Active or Released

📜 Edit History

Scroll down to see a full timeline of every field change — who changed it, when, and the old vs new value.

🖨️ Printing the Barcode

  1. 1Open the item's detail page.
  2. 2Click the 🖨️ Print button near the barcode.
  3. 3A print-optimized label appears with barcode, ticket no, customer name, weight, and karatage.
  4. 4The printer auto-cuts after printing.
✅
The label is sized for a 58mm thermal receipt printer. Make sure the correct printer driver is installed.

🔎 Scan History

Shows every time this barcode was scanned — creation, audit scans, and verification events — with timestamp and operator.

Section 3

Customers

3.1  Browsing Customers

Navigate to Customers. Customers are displayed as cards in a grid.

🔍 Search & Filter

  • Search by name or NIC number
  • Filter by Branch using the branch pills at the top
  • Each pill shows the customer count for that branch

🃏 Customer Cards

Each card shows:

  • Customer initials avatar
  • Full name, NIC number
  • Phone and email
  • Assigned branches
  • Number of pawned items

3.2  Creating a Customer  Admin Only

  1. 1Click + New Customer (top-right).
  2. 2Fill in Full Name (required), NIC, Phone, Email, and Address.
  3. 3Optionally assign to one or more Branches.
  4. 4Click Save.
✅
You can also bulk-import customers using the CSV Import button. Download the template to see required columns.

3.3  Customer Detail Profile

Click any customer card to open their profile page.

📊

Statistics

Total items, active count, released count, and total active gold weight

📞

Contact Info

NIC, phone, email, address, member since date

🏷️

Pawned Items

Table of all items with type, weight, date, and status. Click any row to open item detail.

Section 4

Branches

4.1  Creating a Branch  Admin Only

  1. 1Navigate to Branches in the sidebar.
  2. 2Click + New Branch.
  3. 3Enter the Branch Name (e.g. "Head Office", "Kandy Branch").
  4. 4Click Save. The branch card appears immediately.
ℹ️
Branches are used to filter customers and to scope Initial Audits to a specific location.

4.2  Assigning Customers to a Branch  Admin Only

  1. 1On the Branches page, click the Manage Customers button on a branch card.
  2. 2A side drawer opens showing current members and all available customers.
  3. 3Use the search box to find a customer, then click + Add.
  4. 4To remove, click the ✕ next to a customer's name.
✅
Customers can belong to multiple branches. The branch assignment is used for filtering the customer list and for creating branch-specific initial audits.

4.3  Deleting a Branch  Admin Only

⚠️
Deleting a branch removes all customer-branch assignments for that branch. Customer records themselves are not deleted.
  1. 1Click the 🗑️ Delete button on a branch card.
  2. 2Confirm the action in the dialog.

Standard Audit (Barcode Scan)

What is a Standard Audit?

A standard audit physically verifies all Active items by scanning their barcodes one by one. It reveals which items are present (Found), missing (Missing), or unknown (Unknown).

5.1  Starting a Standard Audit  Admin Only

  1. 1Go to Audits and click New Audit (navy button, top-right).
  2. 2Optionally enter notes about this audit session.
  3. 3Click Start Audit. The audit detail page opens immediately.
ℹ️
The system snapshots all currently active items at the moment the audit is created. Only those items appear as expected.

5.2  Scanning Items

The scanner input is automatically focused when you open the audit. Simply scan each barcode label with a barcode reader.

🖥️ Scanner Input

  • The cursor is always in the scan input box — no need to click
  • After each scan the box clears automatically
  • The scanned item moves to the top of the list with a status badge
  • You can also type a barcode manually and press Enter

⚡ Live Stats Bar

The top of the page shows running counts:

  • ✓ Found — scanned and matched
  • ✗ Missing — expected but not yet scanned
  • ? Unknown — scanned but not in audit list

5.3  Scan Statuses Explained

StatusMeaningAction
✓ FOUND Item was scanned and matched an expected item No action required
✗ MISSING Item was expected but its barcode was not scanned Investigate. Can be released after finalization if confirmed missing.
? UNKNOWN Barcode was scanned but it does not belong to this audit's snapshot Verify the item. It may have been added after the audit started.
✏️
Section 5 (cont.)

Corrections, Finalize & Bulk Release

5.4  Corrections & Remarks During Scanning

Click any scanned item row to expand its detail panel.

📝 Adding Remarks

  • Type in the Remarks text area
  • Remarks save automatically when you click away (on-blur)
  • Useful for noting physical condition of an item

✏️ Correcting Item Values

  • Click Edit to correct Item Type or Net Weight
  • All corrections are permanently recorded with timestamp and operator
  • The correction history panel shows all past changes across all audits
🔒
Corrections and remarks are disabled once an audit is finalized.

5.5  Finalizing the Audit  Admin Only

⚠️
Finalization is irreversible. Once finalized, no edits can be made to the audit.
  1. 1Click Finalize Audit (bottom of the page).
  2. 2Confirm in the dialog. All unscanned active items are marked MISSING.
  3. 3The audit status changes to Finalized.

5.6  Bulk Release of Missing Items  Admin Only

After finalization, items confirmed as missing can be released (removed from active stock).

  1. 1In the finalized audit, tick the checkboxes next to MISSING items.
  2. 2Use Select All Missing to check all at once.
  3. 3Click Release Selected.
  4. 4A two-step confirmation modal shows: first review the list, then click Confirm Release.
  5. 5Released items are marked Released and removed from future audit snapshots.
ℹ️
Only items that are still Active can be released. Already released items are shown greyed-out and cannot be selected.

Initial Audit (Inventory Review)

What is an Initial Audit?

An initial audit is a manual review of your existing inventory — no barcode scanning required. It is used to verify and correct item details (weights, types, karatage) by physically inspecting each item. Items are grouped by ticket number so an operator can review one ticket at a time.

6.1  Creating an Initial Audit  Admin Only

  1. 1Go to Audits and click Initial Audit (purple button, top-right).
  2. 2In the drawer, set optional filters:
FilterDescription
Date From / Date ToOnly include items with pawn dates in this range
BranchOnly include items belonging to customers in the selected branch
NotesInternal notes for this audit session
  1. 3Click Start Initial Audit. You are taken to the audit detail page.
✅
Leave filters blank to include all active items. Filtering by branch or date helps break a large inventory into manageable sessions.

Section 6 (cont.)

Reviewing & Editing Items

6.2  Reviewing Ticket Groups

Items are displayed grouped by ticket number. Each group row shows: Ticket No, Customer Name, Pawn Date, item count, and total net weight.

Progress Bar

At the top of the page a progress bar tracks how many tickets have been marked complete. The bar turns green when 100% of tickets are reviewed — the Finalize button activates at this point.

  1. 1Click any ticket row (or the ▶ chevron) to expand it and see individual items.
  2. 2Review each item — verify the type, weights, and karatage match the physical item.

6.3  Editing Items  Admin Only

  1. 1Expand a ticket group and then expand an individual item.
  2. 2Click ✏️ Edit.
  3. 3Correct any of the following fields:
Item Type
e.g. ring → bracelet
Net Weight
in grams (3dp)
Gross Weight
optional
Karatage
e.g. 22, 18, 14
Remarks
free text notes
  1. 4Click 💾 Save. The change is recorded in the item's edit history.
ℹ️
Every edited field is captured in the audit report. The report shows the original system value alongside the audited corrected value in separate columns.

Section 6 (cont.)

Adding Items to a Ticket

6.4  Adding a New Item to an Existing Ticket  Admin Only

Sometimes one physical article may need to be recorded as two separate items (e.g. a pair of earrings entered as a single record). Use Add Item to Ticket to add the extra item directly within the audit.

  1. 1Expand the relevant ticket group.
  2. 2In the expanded footer, click + Add Item to Ticket (blue button).
  3. 3An inline form appears. Fill in:
FieldRequiredNotes
Item Type✅ YesSelect from pill buttons (ring, chain, bracelet, etc.)
Net Weight (g)✅ YesMinimum 0.001g
Gross Weight (g)NoOptional
Karatage (K)No1–24
RemarksNoFree text
  1. 4Click Add Item to Ticket. The new item appears in the ticket immediately.
✅
The new item inherits the ticket's customer and pawn date automatically. A new barcode is generated for it behind the scenes.
🌟
Items added during an audit are clearly marked in the report with a green "Added in Audit" column showing YES, and highlighted in green in the PDF export.
✅
Section 6 (cont.)

Marking Complete & Finalizing

6.5  Marking a Ticket as Complete

After physically reviewing all items in a ticket group, mark it as complete to track your progress.

  1. 1Expand a ticket group.
  2. 2At the bottom of the expanded footer, click "Done — Mark Ticket Complete" (green button).
  3. 3The ticket row disappears from the pending list and moves to completed.
  4. 4The progress bar at the top increments.
ℹ️
You can also click the ✓ checkmark icon in the ticket row (7th column) to quickly mark a ticket without expanding it.

Progress States

60%
100% ✓

Purple bar while in progress. Turns green at 100% — Finalize button becomes active.

6.6  Finalizing the Initial Audit  Admin Only

⚠️
Finalization is irreversible. All editing is locked permanently after finalization.
  1. 1Ensure all tickets are marked complete (progress bar = 100%).
  2. 2Click Finalize Audit (header, top-right). The button is only enabled when all tickets are done.
  3. 3A confirmation modal appears showing a list of all completed ticket numbers.
  4. 4Read the confirmation word shown in the modal (e.g. "EAGLE").
  5. 5Type the word exactly in the input box.
  6. 6Click Confirm Finalize (or press Enter).

Section 6 (cont.)

Downloading Audit Reports

6.7  Excel & PDF Reports

Both before and after finalization you can download a full report of the audit items. Two buttons appear in the audit header:

📗 Download Excel

Green button — downloads an .xlsx file with two sheets:

  • Audit Info — metadata (date, branch filter, totals, edited/added counts)
  • Audit Items — full data table with all columns

📕 Download PDF

Red button — downloads a landscape A4 .pdf with:

  • Title, audit date, branch, date range, status in header
  • Colour-coded legend explaining column colours
  • Full data table with highlighting
  • Page numbers in footer

Report Column Reference

Item Type✅ YesSystem value (before audit)
Audited Item TypeOnly if any type was editedCorrected value entered during audit

Audited Net Wt (g)Only if any weight was editedCorrected weight
Gross Wt (g)✅ YesSystem gross weight
Audited Gross Wt (g)Only if any gross weight was editedCorrected gross weight

Audited KaratageOnly if any karatage was editedCorrected karatage

Section 7

Reports & Activity Log

7.1  Reports Page

Navigate to Reports for system-wide analytics. This page provides summary statistics and downloadable data exports.

📈 Summary Statistics

  • Total active items and total weight in stock
  • Total customers and branches
  • Items released this month
  • Audit completion rate

📥 CSV Downloads

  • Items CSV — all item records with all fields
  • Customers CSV — all customer records
  • Use these for external reporting or backup

7.2  Activity Log

Navigate to Activity Log to see a chronological record of all significant actions performed in the system.

Events Tracked

  • Item created / edited / released
  • Customer created / updated
  • Branch created / deleted
  • Audit started / finalized
  • Barcode scanned
  • CSV import completed
ℹ️
Each log entry shows the action type, the entity affected, the operator who performed it, and a precise timestamp. Activity logs cannot be deleted.
